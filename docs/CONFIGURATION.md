# Crux Configuration Reference

## Configuration Files

### `.mcp.json` (project root)

MCP server configuration for Claude Code. Defines which servers to start and their environment.

```json
{
  "mcpServers": {
    "crux": {
      "type": "stdio",
      "command": "<repo>/.venv/bin/python",
      "args": ["-m", "scripts.lib.crux_mcp_server"],
      "env": {
        "CRUX_PROJECT": "<project-dir>",
        "CRUX_HOME": "<home-dir>",
        "PYTHONPATH": "<repo-dir>"
      }
    }
  }
}
```

### `config.json` (project root)

Project-level Crux configuration.

| Key | Type | Default | Description |
|-----|------|---------|-------------|
| `default_mode` | string | `"build-py"` | Mode activated on session start |
| `digest_cadence` | string | `"daily"` | How often to generate digests |

### `.claude/settings.local.json`

Claude Code hooks and permissions. Generated by `crux adopt` or `setup.sh`.

| Section | Content |
|---------|---------|
| `permissions.allow` | Bash patterns allowed without confirmation |
| `hooks.SessionStart` | Injects Crux context on session start |
| `hooks.PostToolUse` | Logs interactions after each tool use |
| `hooks.UserPromptSubmit` | Captures user messages for correction detection |
| `hooks.Stop` | Logs session end |

### `pyproject.toml`

Pytest and coverage configuration. Key settings:

| Setting | Value | Purpose |
|---------|-------|---------|
| `testpaths` | `["tests"]` | Where pytest looks for tests |
| `fail_under` | `100` | Coverage enforcement threshold |

### `setup.sh`

Interactive installer (1500+ lines). Handles:
- Tool selection (Claude Code / OpenCode / Both)
- Python venv creation and dependency installation
- MCP server configuration
- Symlink creation for modes, plugins, tools, commands
- Ollama model setup (OpenCode path)

### `.crux/marketing/config.json`

Build-in-public pipeline configuration. See BIP tools in API.md.

## Environment Variables

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `CRUX_PROJECT` | No | `cwd` | Project directory for MCP server |
| `CRUX_HOME` | No | `$HOME` | User home for cross-project state |
| `PYTHONPATH` | Yes (MCP) | — | Must include crux repo root |
| `CRUX_AUDIT_BACKEND` | No | `ollama` | Audit backend (ollama/anthropic/openai/subagent) |
| `CRUX_ANTHROPIC_API_KEY` | No | — | API key for Anthropic audit backend |
| `CRUX_OPENAI_API_KEY` | No | — | API key for OpenAI audit backend |
| `CRUX_ADVERSARIAL_BACKEND` | No | — | Override backend for adversarial audits |
| `CRUX_ADVERSARIAL_MODEL` | No | — | Model for adversarial audit |

## Directory Structure

### `.crux/` (per-project)

```
.crux/
├── knowledge/              # Project-specific knowledge entries
├── corrections/            # Correction log (corrections.jsonl)
├── sessions/               # Session state (state.json)
├── scripts/session/        # Session scripts (pre-promotion)
├── scripts/archive/        # Archived scripts
├── context/                # Project context (PROJECT.md)
├── analytics/              # Conversations, interactions, digests
└── marketing/              # BIP config, state, drafts
```

### `~/.crux/` (per-user)

```
~/.crux/
├── knowledge/shared/       # Cross-project knowledge
├── modes/                  # Canonical mode definitions
├── corrections/            # Cross-project correction patterns
├── analytics/              # User-level analytics
└── projects/               # Registered project registry
```
